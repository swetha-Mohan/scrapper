    • Introduction

    This project is built as part of the three hour "Programming Paradigms" assignment at ProtoSem.

    The assignment's objective is to scrape data from the internet, store it in a database, build an API interface for the database using Flask and to use JavaScript to render the data retrieved from the API as a table.

    Due to the time constraint, a rapid prototyping approach was adopted and the outcomes of several team members were stitched together. This resulted in some interesting problems during integration, a case where this is blatantly evident is the fact that the scraper.py file chooses to directly connect with the database to write data, rather than use the insert functionality that the API implements.

    Application Logic

    The application first scrapes data (needs to be invoked by sending a GET request [Refer: /docs]) and stores it into a MongoDB database running in the cloud (Read: Atlas). Naturally, it goes without saying that we have no idea when the owner will choose to delete the instance; so this may break the entire application at any time.

    The Flask API then connects to the MongoDB instance; implementing methods for CRUD (documented in /docs). The interesting fact here is that the CRUD API implemented via Flask is general-purpose and will work for any application; given that the connection string is replaced (Whoo-hoo!). But it is in no way comprehensive and requires a lot of work to actually be reliable. But it was an interesting exercise to implement a general-purpose API nonetheless.

    The .js script in /templates/table.html then connects to the Flask API and using the /getAll method; gets all the data and renders it in an elegant table format; styled by the BootStrap lords.
